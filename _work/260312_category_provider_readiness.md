# Category Model: Provider Readiness

**Date**: 2026-03-12  
**Goal**: Prepare Category model to replace Organization for data providers  
**Status**: Proposal - Ready for Implementation

---

## Executive Summary

This document outlines the changes needed to make the `Category` model capable of replacing the `Organization` model for data providers (OSM, SAC, etc.). The approach focuses on minimal, targeted additions rather than 1:1 field mapping.

**Key Principles**:
- ✅ Use existing `symbol_*` fields instead of dedicated logo field
- ✅ Add `CategoryExternalLink` M2M for flexible URL management
- ✅ Use `url`/`urls` for field names (not `link`/`links`)
- ✅ Minimal use of `extra` JSON - only for truly dynamic metadata
- ✅ Add template support + rendering to ExternalLink model
- ✅ Enable short/long form names for all categories (useful for brands too)

**New Fields Required**: 2
- `is_public` - Boolean flag on Category
- `label` - TranslationField (short form) on Category

**New Model Required**: 1
- `CategoryExternalLink` - M2M through model with `extra` JSON field

**Changes to ExternalLink**: 2
- `is_template` - Boolean flag to disable health checks
- `render()` - Method to render template URLs with variables

## 2. CategoryExternalLink Model (NEW)

Create a many-to-many through model for associating external links with categories.

### 2.1 Model Definition

```python
# File: server/apps/categories/models.py (or new file _associations.py)

class CategoryExternalLink(models.Model):
    """
    Association between categories and external links (URLs).

    Allows categories to have multiple URLs (homepage, documentation, API docs, etc.)
    with ordering and optional metadata per association.

    Use cases:
    - Provider homepage: provider.osm → "https://www.openstreetmap.org"
    - API documentation: provider.osm → "https://wiki.openstreetmap.org/wiki/API"
    - Dynamic link: provider.osm → "https://osm.org/{{type}}/{{id}}" (template)
    - Brand website: brand.migros → "https://www.migros.ch"
    """

    category = models.ForeignKey(
        "categories.Category",
        on_delete=models.CASCADE,
        related_name="external_link_associations",
        verbose_name=_("Category"),
    )

    external_link = models.ForeignKey(
        "external_links.ExternalLink",
        on_delete=models.CASCADE,
        related_name="categories",
        verbose_name=_("External Link"),
    )

    order = models.PositiveSmallIntegerField(
        default=0,
        verbose_name=_("Order"),
        help_text=_("Display order (lower values appear first)")
    )

    # NEW: Metadata for this specific association
    extra = models.JSONField(
        default=dict,
        blank=True,
        verbose_name=_("Extra Metadata"),
        help_text=_(
            "Additional metadata for this association. "
            "Example: {'variables': ['type', 'id'], 'description': 'OSM edit link'}"
        )
    )

    class Meta:
        db_table = "categories_category_external_link"
        verbose_name = _("Category External Link")
        verbose_name_plural = _("Category External Links")
        ordering = ["order", "id"]
        constraints = [
            models.UniqueConstraint(
                fields=["category", "external_link"],
                name="unique_category_external_link"
            )
        ]
        indexes = [
            models.Index(fields=["category", "order"]),
            models.Index(fields=["external_link"]),
        ]

    def __str__(self) -> str:
        return f"{self.category} → {self.external_link.url}"
```

### 3.2 Add render() Method

Add method to render template URLs with variables:

```python
class ExternalLink(TimeStampedModel):
    # ... existing fields ...

    def render(self, **variables) -> str:
        """
        Render this URL if it's a template, otherwise return as-is.

        Args:
            **variables: Template variables (e.g., id, type, lang)

        Example:
            >>> link = ExternalLink(url="https://osm.org/{{type}}/{{id}}", is_template=True)
            >>> link.render(type="node", id=123456)
            'https://osm.org/node/123456'

        Returns:
            Rendered URL string, or original URL if not a template
        """
        if not self.is_template:
            return self.url

        from jinja2 import Environment

        try:
            template = Environment().from_string(self.url)
            return template.render(**variables)
        except Exception as e:
            # Log error but return empty string on failure
            import logging
            logger = logging.getLogger(__name__)
            logger.warning(f"Failed to render template URL: {self.url}, error: {e}")
            return ""
```

### 6.1 Category Schema

```python
# File: server/apps/categories/schema.py

from ninja import Schema
from typing import Optional, List

class CategoryOut(Schema):
    """Output schema for Category in API."""

    identifier: str
    slug: str
    name: str
    label: Optional[str] = None  # NEW: Include label
    description: Optional[str] = None
    parent_slug: Optional[str] = None
    order: int
    is_active: bool
    is_public: bool  # NEW: Include public flag
    color: str

    # Symbol URLs (if available)
    symbol_detailed_url: Optional[str] = None
    symbol_simple_url: Optional[str] = None
    symbol_mono_url: Optional[str] = None

    # URLs (fetched from CategoryExternalLink associations)
    urls: Optional[List[str]] = None

    @staticmethod
    def resolve_urls(obj):
        """Include URLs for this category."""
        return [url.url for url in obj.urls.filter(is_active=True)]
```

### 6.2 API Endpoint for Provider Categories

```python
# File: server/apps/categories/api.py

from ninja import Router
from typing import List
from .models import Category
from .schema import CategoryOut

router = Router(tags=["categories"])

@router.get("/providers", response=List[CategoryOut])
def list_providers(request):
    """
    Get all data provider categories.

    Returns only public, active categories under the 'provider' parent.
    """
    providers = Category.objects.filter(
        parent__slug="provider",
        is_public=True,
        is_active=True
    ).select_related("parent").prefetch_related("links")

    return list(providers)

@router.get("/brands", response=List[CategoryOut])
def list_brands(request):
    """
    Get all brand categories.

    Returns only public, active categories under the 'brand' parent.
    """
    brands = Category.objects.filter(
        parent__slug="brand",
        is_public=True,
        is_active=True
    ).select_related("parent")

    return list(brands)
```

## 9. Summary of Changes

### 9.1 Models

| Model | Change | Reason |
|-------|--------|--------|
| `Category` | Add `label` field (translatable) | Short form for providers/brands (e.g., "OSM") |
| `Category` | Add `is_public` field | Control visibility in API |
| `Category` | Add helper methods | Easy access to provider metadata in `extra` |
| `CategoryExternalLink` | New model | Many-to-many for flexible link management |
| `ExternalLink` | Add `is_template` field | Skip health checks for URL templates |

### 9.2 What Goes Where

| Feature | Solution | Example |
|---------|----------|---------|
| Logo | Use `symbol_simple` | Upload SVG symbol for provider |
| Short name | Use `label` field | "OSM" |
| Full name | Use `name` field | "OpenStreetMap" |
| Homepage | `CategoryExternalLink` + ExternalLink | Regular URL (is_template=False) |
| API docs | `CategoryExternalLink` + ExternalLink | Multiple URLs per category |
| Dynamic links | `CategoryExternalLink` + ExternalLink | Template URL (is_template=True) |
| Attribution | ExternalLink description | Or in CategoryExternalLink.extra if needed |
| Colors | Use `color` field | Existing field |
| Order | Use `order` field | Existing field |
| Status | Use `is_active`, `is_public` | Existing + new field |
| Extra metadata | Use `extra` on Category or CategoryExternalLink | Only when truly needed |

### 9.3 Benefits

✅ **Minimal changes**: Only 2 new fields on Category, 1 field + 1 method on ExternalLink  
✅ **Flexible URLs**: Multiple URLs per category with ordering via CategoryExternalLink  
✅ **Template rendering**: `ExternalLink.render()` method for dynamic URLs  
✅ **No health checks for templates**: `is_template` flag skips checking  
✅ **Reuse symbols**: Existing symbol infrastructure for logos  
✅ **Association metadata**: `CategoryExternalLink.extra` for URL-specific config  
✅ **Minimal extra usage**: Only use JSON fields when truly dynamic  
✅ **Hierarchical**: Can nest providers (e.g., provider.osm.mapcomplete)  
✅ **Brand support**: Same pattern works for brands with label field
